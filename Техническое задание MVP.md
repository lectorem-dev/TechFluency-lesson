# Техническое задание MVP платформы онлайн-обучения

## Цель

Разработать веб-платформу для прохождения одного обучающего курса с последовательным открытием уроков, тестированием и выдачей сертификата после успешного завершения.

---

# Технологический стек

## Backend

- Java 21
- Spring Boot
- Spring Security
- JWT Authentication
- PostgreSQL
- Liquibase
- Lombok

## Frontend

- React
- TypeScript
- Vite

## Инфраструктура

- Docker
- Docker Compose
- Swagger/OpenAPI

---

# Роли пользователей

## ADMIN

Доступ к административной панели.

Возможности:

- создание пользователя;
- редактирование пользователя;
- архивирование пользователя.

## STUDENT

Доступ к прохождению курса.

---

# Авторизация

На главной странице отображается кнопка входа.

После успешной авторизации:

- ADMIN перенаправляется в административную панель;
- STUDENT перенаправляется в личный кабинет.

Пароли пользователей хранятся в виде хеша.

Для авторизации используется JWT-токен с встроенной ролью пользователя.

---

# Пользователь

Поля пользователя:

- id;
- имя;
- логин;
- пароль;
- статус (ACTIVE / ARCHIVED).

Архивный пользователь не может войти в систему.

---

# Курс

Система поддерживает один курс.

Курс состоит из последовательного набора уроков.

Следующий урок открывается только после успешного прохождения предыдущего.

---

# Урок

Каждый урок содержит:

- название;
- markdown-контент;
- тест.

Markdown отображается непосредственно на странице урока.

---

# Загрузка контента

Контент курса хранится в файловой системе.

При запуске приложения выполняется импорт:

- markdown-файлов уроков;
- JSON-файлов тестов.

После импорта данные сохраняются в PostgreSQL.

Редактирование курса через интерфейс не предусмотрено.

---

# Тестирование

Тест каждого урока описывается JSON-файлом.

Поддерживаются вопросы с одним правильным вариантом ответа.

Для каждого урока задается собственный проходной балл.

При прохождении теста:

- варианты ответов перемешиваются случайным образом;
- производится расчет результата.

Если проходной балл не достигнут:

- попытка считается неуспешной;
- ответы сбрасываются;
- тест начинается заново.

Если проходной балл достигнут:

- урок считается завершенным;
- открывается следующий урок.

Пользователь может повторно проходить уже завершенные уроки и тесты.

Если пользователь покидает страницу теста до завершения, прогресс попытки не сохраняется.

---

# Личный кабинет студента

Отображается:

- имя пользователя;
- название курса;
- текущий прогресс прохождения курса;
- список уроков с информацией о статусе прохождения.

---

# Завершение курса

После успешного прохождения последнего урока и теста пользователю становится доступна страница завершения курса.

На странице отображается поздравление и кнопка скачивания сертификата.

---

# Сертификат

Сертификат формируется в формате PDF.

Содержит:

- ФИО пользователя;
- название курса;
- дату завершения;
- уникальный номер сертификата.

Сертификат может скачиваться неограниченное количество раз.

---

# Ограничения MVP

- один курс;
- один язык интерфейса (русский);
- отсутствует регистрация пользователей;
- отсутствует восстановление пароля;
- отсутствует аналитика и статистика;
- отсутствует редактирование курса через интерфейс;
- отсутствует логирование пользовательских действий;
- отсутствуют автоматические тесты.
